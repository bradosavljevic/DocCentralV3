# 12 - Security And Permissions

## Core model

- Users have Read permissions on SharePoint.
- Write operations are executed by Power Automate service account.
- App does not rely on user Edit rights.

## Organization unit permissions

Optional feature:
- users see documents for their organizational unit groups
- users may belong to multiple organizational units
- process participants retain visibility of documents they participated in

## Role model

Future placeholder roles:
- Admin
- Consultant
- Registrar
- Approver
- Archivist
- Viewer

Current version does not need full role implementation.
