## Filter expression for Get items

Replace UNKNOWN column names before building.

**Option A — dedicated year column:**
```
UNKNOWN_Stanje eq 'Arhivirano' and UNKNOWN_Year eq <varYear>
```

**Option B — date range filter:**
```
UNKNOWN_Stanje eq 'Arhivirano' and UNKNOWN_FilingDate ge '<varYear>-01-01T00:00:00Z' and UNKNOWN_FilingDate lt '<varYearPlusOne>-01-01T00:00:00Z'
```

Where `varYearPlusOne` = `add(variables('varYear'), 1)`

**Constructing the date range strings:**
```
[Compose: Build_YearStart]
concat(string(variables('varYear')), '-01-01T00:00:00Z')

[Compose: Build_YearEnd]
concat(string(add(variables('varYear'), 1)), '-01-01T00:00:00Z')
```

Then use these in the filter expression.

## Sort order

Preferred: by DelovodniBroj numerically (if a separate numeric counter column exists — UNKNOWN).
Fallback: by FilingDate ascending, then by ID.

If DelovodniBroj is stored as text (`"5/2026"`), alphabetic sort is incorrect (`"10/2026"` sorts before `"9/2026"`). Use a numeric counter column or FilingDate as the sort key instead.

OData `$orderby` expression (after column names confirmed):
```
UNKNOWN_NumericCounter asc
```
or:
```
UNKNOWN_FilingDate asc, ID asc
```

## HTML escaping per field

Applied to all text fields before insertion into HTML. No built-in function in Power Automate.

```
[Compose: Escape_<FieldName>]
replace(
  replace(
    replace(
      replace(
        if(empty(outputs('Get_CurrentDoc')?['UNKNOWN_<Field>']), '', string(outputs('Get_CurrentDoc')?['UNKNOWN_<Field>'])),
        '&', '&amp;'
      ),
      '<', '&lt;'
    ),
    '>', '&gt;'
  ),
  '"', '&quot;'
)
```

Apply to: DelovodniBroj, Subject, PartnerNazivSnapshot, DocumentType, ArhivskiZnak, Arhivirao.
Dates are formatted with `formatDateTime()` — no HTML escaping needed for date strings.

## Pagination — @odata.nextLink handling

When using Send an HTTP Request for subsequent pages:
- Response is in OData verbose format: `body(...)?['d']?['results']` for items
- NextLink: `body(...)?['d']?['__next']`

Confirm the correct JSON path after testing the first paginated request.
