# Archive book output specification

## Purpose

Defines the expected structure, content, and format of the archive book (`Arhivska knjiga`)
generated by `CF_DocCentralV3_GenerateArchiveBookPdf`.

This document is used as the design reference for the flow's content composition step.

## Output format

**v1 target**: HTML file saved to the `Exports` SharePoint library.
HTML can be opened in a browser and printed to PDF by the user.

**Future**: True PDF generation via Word template + OneDrive conversion, pending
connector and license confirmation. See `CF_DocCentralV3_GenerateArchiveBookPdf-design.md`
for the technical options.

## File naming

```
ArhivskaKnjiga_{year}_{YYYYMMDD_HHmmss}.html
```

Example:
```
ArhivskaKnjiga_2026_20261215_083012.html
```

Saved in root of the `Exports` library (`EV_DocCentralV3_docExports`).

## Document header

The archive book header (top of the document) must include:

| Field | Source | Notes |
|---|---|---|
| Organization name | App Config (UNKNOWN key) | Placeholder `[Naziv organizacije]` if not configured |
| Title | Static text | `ARHIVSKA KNJIGA` |
| Registry year | Input `year` parameter | |
| Generated on | `utcNow()` formatted | Format: `dd.MM.yyyy HH:mm` |
| Total document count | Count of rows | |

Example header layout:
```
[Naziv organizacije]
ARHIVSKA KNJIGA ZA {year}. GODINU

Datum generisanja: {date}
Ukupan broj dokumenata: {count}
```

## Archive book table columns

The body of the archive book is a table with one row per archived document.

| # | Column header | Data source | Notes |
|---|---|---|---|
| 1 | R. br. | Sequential row number | Generated by the flow, not from SharePoint |
| 2 | Delovodni broj | `DelovodniBroj` from Svi predmeti | UNKNOWN internal column name |
| 3 | Datum zavođenja | Filing date from Svi predmeti | UNKNOWN column name; format: `dd.MM.yyyy` |
| 4 | Predmet / sadržaj | Title or Subject from Svi predmeti | UNKNOWN column name |
| 5 | Partner | `PartnerNazivSnapshot` from Svi predmeti | UNKNOWN internal column name |
| 6 | Tip dokumenta | `DocumentType` from Svi predmeti | UNKNOWN internal column name |
| 7 | Arhivski znak | `ArhivskiZnak` from Svi predmeti | UNKNOWN internal column name |
| 8 | Datum arhiviranja | `ArchivedAt` from Svi predmeti | UNKNOWN column — may not exist yet |
| 9 | Ko je arhivirao | `ArchivedBy` from Svi predmeti | UNKNOWN column — may not exist yet |

All internal column names are UNKNOWN until Svi predmeti column schema is confirmed.

## Sort order

Rows are ordered by `DelovodniBroj` ascending.
If `DelovodniBroj` is stored as text (e.g. `"1/2026"`, `"42/2026"`), numeric sort must be
applied to avoid lexicographic ordering (`"10/2026"` before `"9/2026"`).

Recommended approach: sort by a numeric counter column if one exists separately from the
formatted text value. UNKNOWN — depends on Svi predmeti schema.

## HTML structure (v1)

```html
<!DOCTYPE html>
<html lang="sr">
<head>
  <meta charset="UTF-8">
  <title>Arhivska knjiga {year}</title>
  <style>
    body { font-family: Arial, sans-serif; font-size: 11pt; margin: 20mm; }
    h1 { text-align: center; font-size: 14pt; }
    .subtitle { text-align: center; font-size: 12pt; margin-bottom: 10px; }
    .meta { margin-bottom: 20px; font-size: 10pt; }
    table { width: 100%; border-collapse: collapse; font-size: 10pt; }
    th { background-color: #f0f0f0; border: 1px solid #888; padding: 4px 6px; text-align: left; }
    td { border: 1px solid #aaa; padding: 3px 6px; vertical-align: top; }
    tr:nth-child(even) { background-color: #fafafa; }
    @media print { body { margin: 10mm; } }
  </style>
</head>
<body>
  <h1>{OrgName}</h1>
  <div class="subtitle">ARHIVSKA KNJIGA ZA {year}. GODINU</div>
  <div class="meta">
    Datum generisanja: {generatedDate}<br>
    Ukupan broj dokumenata: {totalCount}
  </div>
  <table>
    <thead>
      <tr>
        <th>R. br.</th>
        <th>Delovodni broj</th>
        <th>Datum zavođenja</th>
        <th>Predmet / sadržaj</th>
        <th>Partner</th>
        <th>Tip dokumenta</th>
        <th>Arhivski znak</th>
        <th>Datum arhiviranja</th>
        <th>Ko je arhivirao</th>
      </tr>
    </thead>
    <tbody>
      {rows}
    </tbody>
  </table>
</body>
</html>
```

Each `{rows}` row follows the pattern:
```html
<tr>
  <td>{rowNumber}</td>
  <td>{delovodniBroj}</td>
  <td>{filingDate}</td>
  <td>{subject}</td>
  <td>{partnerNazivSnapshot}</td>
  <td>{documentType}</td>
  <td>{arhivskiZnak}</td>
  <td>{archivedAt}</td>
  <td>{archivedBy}</td>
</tr>
```

## Encoding

File encoding: UTF-8 with BOM (`﻿`) prepended.
This ensures correct rendering of Serbian characters (č, ć, š, ž, đ) when opened in Windows.

## Pagination for large sets

If the document count exceeds 5000 items (SharePoint Get items limit), the flow must
use Do-Until pagination with `@odata.nextLink` to retrieve all pages and compose the
complete HTML table before saving the file.

## Exports library

The generated file is saved in the root of the `Exports` library
(`EV_DocCentralV3_docExports`). No subfolders.

No automatic cleanup of old export files. Manual cleanup is admin responsibility.
