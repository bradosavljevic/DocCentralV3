# Test cases: CF_DocCentralV3_LogEvent

## How to test

Test each case by calling the flow manually via the **Test** button in Power Automate
(use the "With my recent trigger values" option, or manually input test values).

Alternatively, call the flow from a temporary instant cloud flow created for testing,
using the **Run a Child Flow** action with the required inputs.

After each test, verify the result in:
1. The flow run history (Power Automate → My Flows → CF_DocCentralV3_LogEvent → Run history).
2. The AuditLog SharePoint list (open list and check the latest item).

---

## TC-001: Happy path — all fields provided

**Purpose:** Verify that a complete log entry is created successfully when all fields are provided.

**Input:**
```json
{
  "eventType": "CreateDocument",
  "eventCategory": "Document",
  "severity": "Info",
  "status": "Started",
  "correlationId": "test-correlation-001",
  "flowName": "CF_DocCentralV3_CreateDocument",
  "flowRunId": "08585000000000000001",
  "source": "CloudFlow",
  "userEmail": "test.user@organizacija.rs",
  "userDisplayName": "Test Korisnik",
  "documentItemId": 0,
  "delovodniBroj": "1/2026",
  "reservedNumberId": 0,
  "partnerId": 0,
  "message": "Pokretanje zavođenja dokumenta.",
  "detailsJson": "{\"test\": true}",
  "errorCode": "",
  "errorMessage": "",
  "requestPayloadJson": "{\"documentType\": \"Ulazni\"}",
  "responsePayloadJson": ""
}
```

**Expected flow response:**
```json
{
  "success": true,
  "auditLogItemId": <any positive integer>,
  "correlationId": "test-correlation-001",
  "message": ""
}
```

**Expected AuditLog item:**
- Title = `CreateDocument`
- EventType = `CreateDocument`
- EventCategory = `Document`
- Severity = `Info`
- Status = `Started`
- CorrelationId = `test-correlation-001`
- FlowName = `CF_DocCentralV3_CreateDocument`
- UserEmail = `test.user@organizacija.rs`
- DelovodniBroj = `1/2026`
- Message = `Pokretanje zavođenja dokumenta.`
- CreatedAt = timestamp within a few seconds of test execution

**Pass criteria:** Flow run shows green. AuditLog item exists with correct values.

## TC-003: Empty correlationId — auto-generation

**Purpose:** Verify that the flow generates a new GUID when `correlationId` is empty.

**Input:** Same as TC-002 (correlationId = empty string).

**Expected:** Response `correlationId` is a valid GUID (36 chars, format `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`).
AuditLog item `CorrelationId` column contains the same GUID.

**Pass criteria:** CorrelationId in response matches CorrelationId in the AuditLog item.
Both are valid GUIDs.

## TC-005: Invalid Choice value

**Purpose:** Verify that the Catch scope handles a SharePoint validation error gracefully when an invalid Choice value is passed.

**Input:** Same as TC-001, but with:
```json
"severity": "INVALID_VALUE"
```

**Expected flow response:**
```json
{
  "success": false,
  "auditLogItemId": 0,
  "message": "Audit log write failed.",
  "correlationId": "<correlationId or auto-generated GUID>"
}
```

**Verification:**
- Flow run shows red on the Create item action inside Try scope.
- Catch scope executed successfully.
- No AuditLog item created.
- No exception propagated to the caller (response is structured, not an HTTP error).

**Pass criteria:** Flow returns `success: false` without throwing. Caller receives a valid JSON response.

## TC-006: SharePoint list unavailable (simulated)

**Purpose:** Verify that a SharePoint connectivity error is caught and does not crash the flow.

**Simulation approach:**
- Temporarily set `EV_DocCentralV3_lstAuditLog` to a non-existent list name (e.g. `AuditLogXXX`).
- Run the flow with any valid input.

**Expected:** Same as TC-005 — `success: false`, no exception to caller.

**Restore:** Set `EV_DocCentralV3_lstAuditLog` back to the correct list name after the test.

**Pass criteria:** Flow returns structured failure response. No unhandled exception.

## TC-010: Concurrent calls (two simultaneous runs)

**Purpose:** Verify that two simultaneous calls do not interfere — each creates its own AuditLog item.

**Method:** Trigger two test runs in quick succession (manually or via a temporary parallel flow).

**Expected:** Two separate AuditLog items created, each with correct data. No overwrite. No deadlock.

**Pass criteria:** Two items exist in AuditLog after the test, with distinct IDs and timestamps.

## Post-test cleanup

After completing all test cases:
1. Delete or filter test AuditLog items to keep the list clean (optional — test items are harmless but clutter the view).
2. Restore any environment variable values changed during testing (TC-006).
3. Confirm the flow is in the `DocCentralV3` solution before proceeding to the next flow.
