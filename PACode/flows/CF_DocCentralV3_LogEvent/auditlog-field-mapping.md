# AuditLog field mapping: CF_DocCentralV3_LogEvent

## Purpose

Maps each flow input parameter to the corresponding AuditLog SharePoint list column.
Used as the reference when configuring the **SharePoint — Create item** action in Power Automate.

## Status of column names

**Display names** are confirmed from `data-model/audit-log.md`.
**Internal column names** are UNKNOWN — they must be read from the actual SharePoint list
column settings before building the flow. Do not guess internal names.

To find internal column names:
1. Open the AuditLog list in SharePoint.
2. Go to List Settings → Columns → click each column name.
3. The internal name appears in the URL as `Field=<InternalName>`.

## Title field

The `Title` column (SharePoint's built-in required field) must be populated.
Set it to the same string as the `eventType` input parameter.

Example: if `eventType = "CreateDocument"`, set `Title = "CreateDocument"`.

This makes the list view readable without opening each item.

---

## Choice column handling in Power Automate

When using the **SharePoint — Create item** connector action (standard connector, not HTTP):
- Choice columns appear as text fields in the Power Automate form.
- Set the choice column value by entering the choice option string directly.
- Example: for the `Severity` column, enter `Info`, `Warning`, `Error`, or `Critical`.
- Do not wrap in quotes or use a JSON object — the connector handles serialization.

If the choice value passed by the caller does not match any defined option exactly
(case-sensitive), SharePoint will reject the item creation with a validation error.
The Catch scope will handle this and return a failure response.

---

## Number column handling

For Number columns (`DocumentItemId`, `ReservedNumberId`, `PartnerId`):
- If the caller passes `0` or an empty value, pass `null` to the SharePoint column rather than `0`.
- Passing `0` is valid but may be confusing in audit queries (0 vs not-set).
- Recommended: use a Condition before the Create item action to check if the value is `0`
  or empty, and if so, leave the column unset (null) in the Create item body.

---

## CreatedAt column

`CreatedAt` is set by the flow using `utcNow()`, not taken from the input.
This ensures the timestamp is set at the moment of writing, regardless of caller clock.

If the AuditLog list was created with a column named `Created` (system column),
a custom column named `CreatedAt` must be created separately.
The system `Created` column cannot be written to via the connector.

## Columns not in trigger input

| Display name | Source | Notes |
|---|---|---|
| Title | `eventType` input | Always set to eventType value |
| CreatedAt | `utcNow()` in flow | Set at write time |

| SolutionName | Static `DocCentralV3` | Hardcoded — consistent across all tenants |

---

## Validation before item creation

The flow does not validate Choice values client-side before attempting the Create item action.
SharePoint column validation handles invalid choice values.
Any SharePoint error is caught by the Catch scope and returned as `success: false`.

This is intentional — client-side validation of all choice combinations would duplicate
SharePoint's own constraint enforcement and add unnecessary complexity.
